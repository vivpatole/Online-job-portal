✅ Root Cause of the Error

Your error:

Cannot add or update a child row: a foreign key constraint fails (job_portal.job, CONSTRAINT job_ibfk_1 FOREIGN KEY (posted_by_id) REFERENCES user (id))

It occurs because:

  • The job.posted_by_id is a foreign key referring to user.id.

  • While inserting the job record, the posted_by_id is either NULL or points to a non-existent user ID in the user table.

✅ High-Level Diagnosis

  1. postedBy field in your Job entity is not set properly before saving.

  2. Most likely, userRepository.findByUsername(userDetails.getUsername()) is returning null (no user found), but still you proceed to save.

  3. This leads to setting postedBy to null, which violates the foreign key constraint when inserting into the job table.

✅ Step by Step Solution

✅ Step 1: Ensure Authenticated User Exists

In your controller, verify that the authenticated user is fetched properly.

java
@PostMapping("/add")
public String addJob(@ModelAttribute Job job, @AuthenticationPrincipal UserDetails userDetails) {
    User user = userRepository.findByUsername(userDetails.getUsername());

    if (user == null) {
        throw new RuntimeException("Authenticated user not found in the database");
    }

    job.setPostedBy(user);
    job.setPostedDate(LocalDate.now()); 
    jobRepository.save(job);

    return "redirect:/jobs"; // or any page you want
}

✅ Step 2: Verify the userDetails value

Before fetching the user:

java
System.out.println("Authenticated Username: " + userDetails.getUsername());

If this prints null or empty, then:

  • Your authentication session may not be holding the right UserDetails.

  • Check your Spring Security authentication configuration.

  • Ensure the user is logged in before accessing /add.

✅ Step 3: Ensure that userRepository.findByUsername works

In UserRepository:

java
Optional<User> findByUsername(String username);

Then:

java
Optional<User> userOptional = userRepository.findByUsername(userDetails.getUsername());
if (userOptional.isEmpty()) {
    throw new RuntimeException("User not found for username: " + userDetails.getUsername());
}
User user = userOptional.get();

Make sure your user table has the expected usernames.

Example MySQL Query:

sql
SELECT * FROM user WHERE username = '<username>';

Replace <username> with your logged-in username.

✅ Step 4: Confirm Job Entity Mapping

Make sure your Job entity has:

java
@ManyToOne
@JoinColumn(name = "posted_by_id", nullable = false)
private User postedBy;

And your User entity has:

java
@OneToMany(mappedBy = "postedBy")
private List<Job> jobs;

This ensures proper relationship mapping.

✅ Step 5: Check the user Table Data

Run this in MySQL:

sql
SELECT id, username FROM user;

Ensure that:

  • The username fetched via userDetails.getUsername() matches a username in your DB.

  • The corresponding id is not null.

✅ Step 6: (Optional) Check for Cascade Settings

If you're using cascade on Job entity:

java
@ManyToOne(cascade = CascadeType.PERSIST)
@JoinColumn(name = "posted_by_id", nullable = false)
private User postedBy;

But ideally, don't persist users while saving a job — just reference an existing user.
